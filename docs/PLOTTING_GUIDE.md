# Plotting Guide

The repository is Plotly-first. Most figures are generated by workflow modules
called through `scripts/run_analysis.py`, saved with shared helpers in
`moimpact/plotting.py`, styled by `moimpact/plot_style.py`, and finally copied or
written into the paper snapshot under `paper/images/` when they are used by the
manuscript.

This guide documents every supported way to generate figures and the conventions
to follow when adding or editing plots.

## Figure-generation methods

### 1. Standard full pipeline refresh

Use the end-to-end pipeline when you want the standard proprietary/client refresh
for all nationality slices and the canonical downstream diagnostics:

```bash
conda activate main
bash run_all_pipelines.sh
```

Important controls:

- `MAX_JOBS` controls parallel jobs; default is `4`.
- `RUN_TAG` names the log folder under `out_files/{DATASET_NAME}/logs/`.
- `DISABLE_PLOT_LEGENDS=true` hides legends in figures saved through the shared
  Plotly helper.
- `IMG_OUTPUT_PATH_OVERRIDE=images/runs/{DATASET_NAME}` redirects generated
  figure roots without editing YAML files.
- `MIN_FREE_GB` emits a low-disk warning before starting large runs.

`run_all_pipelines.sh` uses temporary YAML copies and passes them through the
workflow-specific environment variables. It should not edit `config_ymls/*.yml`
in place.

### 2. Individual workflow commands

Use the unified dispatcher when you only need one analysis family:

```bash
conda activate main
python scripts/run_analysis.py --list
python scripts/run_analysis.py <group> <command> --config config_ymls/my_config.yml
```

The dispatcher accepts `--config` and `--config-path` as uniform aliases. For
legacy modules it sets the relevant environment variable before import; for newer
modules it forwards `--config-path` when the workflow accepts it.

Current figure-producing workflows:

static files in `png/` and interactive files in `html/`. Tables, manifests, and
logs usually go under `out_files/{DATASET_NAME}/...`.

### 3. Paper figure runner

Use the paper runner when the goal is to regenerate the figures referenced by
`paper/main.tex` into the stable manuscript snapshot:

```bash
conda activate main
python scripts/run_analysis.py paper figures --list-targets
python scripts/run_analysis.py paper figures --dry-run --targets crowding
python scripts/run_analysis.py paper figures --targets all
```

Default settings live in `config_ymls/paper_figures.yml`.

The runner:

- inspects active `\includegraphics{...}` entries in `paper/main.tex`;
- maps selected figures to the minimal upstream workflow tasks;
- prepares temporary YAML overrides so figures are written under
  `paper/images/` by default;
- writes a run manifest to the paper-figures log folder;
- supports `--targets`, `--figures`, `--list-targets`, `--list-figures`,
  `--dry-run`, `--max-workers`, `--style-mode`, `--paper-style-config`, and
  `--img-output-root`;
- can emit PDF sidecars for Plotly figures when `WRITE_PDF: true` in
  `paper_figures.yml` or when `PLOTLY_WRITE_PDF=true` is set.

Known target groups include `member_stats`, `summary`, `distributions`,
`execution_schedule`, `impact`, `crowding`, `event_study`, `appendix_it`,
`all_main`, and `all`.

For paper figures, make visual changes in the generator, the workflow YAML, or
`config_ymls/paper_figure_styles.yml`. Do not compensate for unreadable figures
with LaTeX-only `\includegraphics` resizing unless the requested change is
explicitly LaTeX-only.

### 4. Programmatic new figures

For new analysis code, prefer Plotly plus the shared output helper:

```python
from pathlib import Path
import plotly.graph_objects as go

from moimpact.plot_style import apply_shared_plotly_style
from moimpact.plotting import make_plot_output_dirs, save_plotly_figure

apply_shared_plotly_style()

fig = go.Figure()
fig.add_trace(go.Scatter(x=[1, 2, 3], y=[2, 1, 3], mode="lines+markers"))
fig.update_layout(xaxis_title="X", yaxis_title="Y")

dirs = make_plot_output_dirs(Path("images/example"), use_subdirs=True)
save_plotly_figure(
    fig,
    stem="example_plot",
    dirs=dirs,
    write_html=True,
    write_png=True,
    strict_png=False,
)
```

Use Matplotlib only for diagnostic/fallback plots that are not naturally
interactive. The current Matplotlib branch is in `crowding eta`; it applies
`apply_matplotlib_style(...)`, uses the non-interactive `Agg` backend, and saves
PNG files directly with `fig.savefig(...)`.

## Shared plotting modules

`moimpact/plot_style.py`

- loads `config_ymls/plot_style.yml` or `PLOT_STYLE_CONFIG`;
- defines the `PlotStyleSpec` dataclass;
- registers and activates the `moimpact_white` Plotly template;
- exposes `load_plot_style(...)`, `apply_shared_plotly_style(...)`,
  `plotly_legend_layout(...)`, and `apply_matplotlib_style(...)`.

`moimpact/plotting.py`

- defines canonical colors: `COLOR_PROPRIETARY`, `COLOR_CLIENT`,
  `COLOR_NEUTRAL`, `COLOR_BAND_PROPRIETARY`, and `COLOR_BAND_CLIENT`;
- exposes `PlotOutputDirs`, `make_plot_output_dirs(...)`,
  `ensure_plot_dirs(...)`, and `save_plotly_figure(...)`;
- exposes size helpers used by workflows with configurable canvases:
  `plotly_figure_size_from_config(...)`, `plotly_layout_size_kwargs(...)`, and
  `plotly_export_size_kwargs(...)`.

`moimpact/paper_figure_styles.py`

- loads `config_ymls/paper_figure_styles.yml` or
  `PAPER_FIGURE_STYLES_CONFIG`;
- supports two modes via `PAPER_FIGURE_STYLE_MODE`: `global` and `per-figure`;
- applies optional stem/path-specific overrides through
  `apply_plotly_paper_figure_style(...)`;
- validates supported keys such as `width`, `height`, font sizes, line widths,
  `showlegend`, colors, band alpha, and margins.

## Output layout and snapshot rules

The canonical helper pattern is:

```python
dirs = make_plot_output_dirs(Path("images/my_analysis"), use_subdirs=True)
```

which implies:

- interactive HTML files in `.../html/`;
- static PNG files in `.../png/`;
- optional PDF sidecars in the same `png/` folder as the PNG.

Use `use_subdirs=False` only when a legacy or one-off output folder must be
flat.

There are two figure roots with different roles:

- `images/{DATASET_NAME}/...` is the generated pipeline output tree.
- `paper/images/...` is the stable LaTeX snapshot consumed by `paper/main.tex`.

When updating paper figures, regenerate from code/config, then copy or write only
publication-ready artifacts into `paper/images/...`. Commit the paper snapshot
with the manuscript change.

## Shared save helper behavior

Canonical export pattern:

```python
html_path, png_path = save_plotly_figure(
    fig,
    stem="my_plot",
    dirs=dirs,
    write_html=True,
    write_png=True,
    write_pdf=False,
    strict_png=False,
)
```

Important behavior:

- `ensure_plot_dirs(...)` is called before writing.
- HTML export includes Plotly JS from CDN by default.
- HTML export includes MathJax from CDN by default, so LaTeX labels render in
  saved HTML.
- PNG/PDF export uses Plotly's static backend, normally `kaleido`.
- `write_pdf=True` or `PLOTLY_WRITE_PDF=true` writes `stem.pdf` beside
  `stem.png`.
- `strict_png=False` allows HTML export to succeed even if static PNG/PDF export
  fails.
- `DISABLE_PLOT_LEGENDS=true` hides legends in all figures saved through this
  helper.
- In `per-figure` paper style mode, the helper applies
  `paper_figure_styles.yml` overrides by output stem before export.

## Style controls and precedence

Global defaults are loaded from `config_ymls/plot_style.yml`; keys omitted from
that file fall back to defaults in `moimpact/plot_style.py`.

Supported global style keys include:

- typography: `FONT_FAMILY`, `FONT_COLOR`, `TICK_FONT_SIZE`,
  `LABEL_FONT_SIZE`, `TITLE_FONT_SIZE`, `LEGEND_FONT_SIZE`,
  `ANNOTATION_FONT_SIZE`;
- colors/background: `THEME_COLORWAY`, `GRID_COLOR`, `AXIS_LINE_COLOR`,
  `BG_COLOR`, legend background/border/position keys;
- geometry/export: `DEFAULT_LINE_WIDTH`, `REFERENCE_LINE_WIDTH`,
  `DEFAULT_MARKER_SIZE`, `EXPORT_WIDTH`, `EXPORT_HEIGHT`, `EXPORT_SCALE`,
  `MATPLOTLIB_DPI`.

Per-figure paper overrides are in `config_ymls/paper_figure_styles.yml` and are
activated by:

```bash
PAPER_FIGURE_STYLE_MODE=per-figure
PAPER_FIGURE_STYLES_CONFIG=config_ymls/paper_figure_styles.yml
```

The paper runner sets these from `config_ymls/paper_figures.yml` when
`STYLE_MODE: per-figure` is active.

Impact-fit canvas dimensions for the paper are controlled from
`config_ymls/paper_figures.yml` via `IMPACT_FIT_FIGURE_WIDTH` and
`IMPACT_FIT_FIGURE_HEIGHT`. The paper runner forwards these values to the
underlying generator configs for the relevant impact-fit figures. Outside the
paper runner, the same keys can be set directly in the workflow YAMLs that
support them, such as `metaorder_computation.yml`, `plot_prop_nonprop_fits.yml`,
and `crowding_impact_analysis.yml`.

Some figure-specific colors, marker sizes, or annotations are still coded inside
individual plotting functions. If a style key appears not to affect a figure,
inspect the generator before assuming the key is global.

## Title handling

Many workflow modules wrap `moimpact.plotting.save_plotly_figure(...)` and call
`fig.update_layout(title=None)` before export. This keeps paper captions as the
main source of title text and prevents duplicate titles in LaTeX composites.

Current title-removing wrappers are used by:

- `metaorders compute`
- `metaorders distributions`
- `metaorders summary`
- `metaorders intraday-impact`
- `metaorders start-event`
- `metaorders start-time`
- `crowding daily`
- `crowding eta`
- `crowding impact`
- `crowding intraday`
- `execution schedule`
- `execution cluster`
- `execution typology`
- `members stats`

If you want an exported file to keep a top-level title, check whether the module
has a local `save_plotly_figure(...)` wrapper and either avoid the wrapper or
make an explicit exception.

## Verification checklist

After changing figure code, config, or this guide:

1. Run the smallest relevant workflow with `--dry-run` when available.
2. Regenerate one representative figure before running the full pipeline.
3. Check that both expected output formats exist (`html/`, `png/`, and optional
   `.pdf`).
4. For paper figures, compile with:

   ```bash
   latexmk -cd -pdf -interaction=nonstopmode -halt-on-error paper/main.tex
   ```

5. Inspect the compiled PDF page when font size or panel readability is the
   requested change.
6. Record command, config, seed/bootstrap settings, and output paths when the
   figure supports paper claims.
